    网络是什么？

    将地理位置不同的多台计算机，通过通信线路连接起来，在网络操作系统，网络管理软件及网络通信协议的管理和协调下，实现资源共享和信息传递的计算机系统。

    • 交换机是组建局域网的，路由器是组建广域网的
    • 路由器有开关，交换机没有开关，插上电直接运行
    • 局域网（LAN）：一组终端设备和由共同的组织管理的用户
    • 广域网（WAN）：连接分布于不同地理位置的 LAN

    分层的优点

    1. 促进标准化工作,允许各个供应商进行开发
    2. 各层间相互独立,把网络操作分成低复杂性单元
    3. 灵活性好,某一层变化不会影响到别层,设计者可专心设计和开发模块功能

    OSI 和 TCP/IP

    • OSI:物理层、数据链路层、网络层、传输层、会话层、表示层、应用层
    • TCP/IP:网络接口层(物理层、数据链路层)、网际互连层(网络层)、传输层、应用层(会话层、表示层、应用层)

    物理层

    • 物理层的主要作用是产生并检测电压发送和接收带有数据的电气信号
    • 物理层是不提供数据的纠错服务的，但是在物理层上能对数据的传输速度作一定的控制，并能监测数据的出错率
    • 物理层传输电气信号的载体我们称之为位流或比特流
    • 物理层设备：屏蔽双绞线（STP）、非屏蔽双绞线（UTP）、光纤、无线、集线器、转换器（调制解调器）、中继器等等

    数据链路层

    • 数据链路层决定数据通讯的机制，差错检测
    • 提供对网络层的服务
    • 合成传输的数据帧（frame）
    • 数据链路层设备：交换机、网桥、网卡（MAC地址）
    • MAC 地址的结构：由48为二进制数组成，通常表示为12个16进制数，前24位是标准组织（IEEE）制定的，后24位是厂家自己制定的节点标识符（00-1C-25-91-65-48或者001C.2591.6548）

    网络层

    • 为网络设备提供逻辑地址
    • 负责数据从远端发送到目的端
    • 负责数据传输的寻径和转发
    • MAC 地址对应的网络为以太网，IP 地址对应的网络为因特网
    • 合成传输的数据包（packet）
    • 网络层设备：路由器

    应用层

    • 应用层的作用主要是为应用软件提供接口，从而使得应用程序能够使用网络服务
    • 应用层协议：http（端口80）、ftp（端口21）、telnet（端口23）、dns（端口53）、smtp（端口25）、rip（端口520）等等

    传输层

    • 传输层负责建立端到端的连接，负责数据在端到端之间的传输
    • 传输层通过端口号区分上层服务
    • 传输控制协议（TCP）：面向连接、可靠传输、流控及窗口机制、例如Web浏览器、电子邮件、文件传输程序
    • 用户数据报协议（UDP）：无连接、不可靠传输、尽力而为的传输、例如域名系统（DNS）、视频流
    • 合成传输的数据段（segment）

    IP 地址分类

    • A 类 1.0.0.1到126.255.255.254 默认掩码 255.0.0.0 == /8
    • B 类 128.0.0.1到191.255.255.254 默认掩码 255.255.0.0 == /16
    • C 类 192.0.0.1到223.255.255.254 默认掩码 255.255.255.0 == /24
    • D 类 224.0.0.1到239.255.255.254（多播也叫组播地址）
    • E 类 240.0.0.1到255.255.255.254（科研用）
    • 主机号全0为网络地址，主机号全1为广播地址
    • 127.0.0.1 为本地环回（loopback）测试地址 用命令 ping 127.0.0.1可以测试网卡是否正常
    • 网络地址（Network）：也可以称为网络号，唯一指定了每个网络。同一个网络中的每台计算机都共享相同的网络地址，并用它作为自己的 IP 地址的一部分。它定义了 IP 地址所属的网段 主机位全为0，表示是否属于同一子网，网络地址相同表示属于同一子网，网络地址不同表示属于不同子网
    • 结点地址（Host）：也可以称为主机地址，是在一个网络中用来标识每台计算机的，它也是唯一的标识符，相对于网络而言，它是用来独立标志指定计算机的
    • 广播地址：主机位全为1，作用是向整个子网发送消息

    子网掩码的作用

    • 用来严格区分网络位和主机位
    • 网络位相同的主机可以直接通信，网路位不同的主机不能直接通信，必须借助第三层设备（路由）
    • 子网掩码使用与 IP 相同的编码格式，子网掩码为1的部分对应于 IP 地址的网络部分，子网掩码为0的部分对应于 IP 地址的主机部分
    • 点分十进制表示法：255.255.0.0 前缀表示法：/16

    私有 IP 地址

    • A 类 10.0.0.0到10.255.255.255
    • B 类 172.16.0.0到172.31.255.255
    • C 类 192.168.0.0到192.168.255.255
    • 私有 IP 地址一般用于公司内部，私有地址可以相同，这会涉及到 NAT

    VLSM

    • 可变长子网掩码（Variable-Length Subnet Masks VLSM）的出现是打破传统的以类（class）为标准的地址划分方法，是为了缓解 IP 地址紧缺而产生的
    • 作用：节约 IP 地址空间
    • 注意：使用 VLSM 时，所采用的路由协议必须能够支持它，这些路由协议包括 RIPv2、OSPF、EIGRP 和 BGP（有类/无类）

    例如：某公司现有一 C 类网段 192.168.1.0/24。目前有以下几个部门：销售部59台电脑、技术部27台电脑、业务部121台电脑、会计部10台电脑。需求：将一个 C 类网段合理分配给如下几个部门。保证地址合理分配
    答：由题目知需求最大的业务部需要121台，其次是销售部，技术部，会计部
    正常 C 类网段的子网掩码为 255.255.255.0 转换为二进制就是 11111111 11111111 11111111 00000000
    后八位为主机号 00000000 变成 10000000 = 128 也就是主机号变成七位可容纳的最大主机数为 2 的 7 次幂 = 128 再减去 广播地址和网络地址为126 符合业务部的需求，所以划分 192.168.1.0/25到192.168.1.127/25 子网掩码是 255.255.255.128，其中192.168.1.0为网络地址，192.168.1.127为广播地址
    后八位为主机号 00000000 变成 11000000 = 192 也就是主机号变成六位可容纳的最大主机数为2 的 6 次幂 = 64 再减去 广播地址和网络地址为 62 符合销售部的需求，所以划分 192.168.1.128/26到192.168.1.191/26 子网掩码是 255.255.255.192，其中192.168.1.128为网络地址，192.168.1.191为广播地址
    后八位为主机号 00000000 变成 11100000 = 224 也就是主机号变成五位可容纳的最大主机数为2 的 5 次幂 = 32 再减去 广播地址和网络地址为 30 符合技术部的需求，所以划分 192.168.1.192/27到192.168.1.223/27 子网掩码是 255.255.255.224，其中192.168.1.192为网络地址，192.168.1.223为广播地址
    后八位为主机号 00000000 变成 11110000 = 240 也就是主机号变成四位可容纳的最大主机数为2 的 4 次幂 = 16 再减去 广播地址和网络地址为 14 符合会计部的需求，所以划分 192.168.1.224/28到192.168.1.239/28 子网掩码是 255.255.255.240，其中192.168.1.224为网络地址，192.168.1.239为广播地址
