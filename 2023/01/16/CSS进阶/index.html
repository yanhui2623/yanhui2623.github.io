    浮动

    • 多个块级元素纵向排列找标准流，多个块级元素横向排列找浮动
    • float: none(默认)/left/right;
    • 浮动的盒子脱离标准流的控制，不再保留原先的位置，下面的会飘上来
    • 多个盒子都设置浮动，它们会一行内显示并且顶端对齐排列，一行位置不够会挤到第二行
    • 任何元素都可以浮动，不管以前是什么模式的元素，添加浮动之后具有行内块元素特性
    • 浮动布局注意点：1.先用标准流的父元素排列上下位置，之后内部子元素采取浮动排列左右位置 2. 一个盒子里有多个盒子，其中一个盒子浮动了，其他兄弟也应该浮动，以防止引起问题

    清除浮动

    由于父级盒子很多情况下，不方便给高度，但是子盒子浮动又不占有位置，最后父级盒子高度为0时，就会影响下面的标准流盒子

    1. 额外标签法 <div style="clear: both"></div> 新添加的元素必须是块级元素
    2. 给父元素添加 overflow: hidden;
    3. :after 伪元素法 给父元素添加 class="clearfix"
    1. 双伪元素清楚浮动 给父元素添加 class="clearfix"

    为什么要清楚浮动？因为父级没高度，子盒子浮动了并且影响了下面的布局我们就应该清楚浮动

    CSS 属性书写顺序

    1. 布局定位属性：display/position/float/clear/visibility/overflow
    2. 自身属性：width/height/margin/padding/border/background
    3. 文本属性：color/font/text-decoration/text-align/vertical-align/white-space/break-word
    4. 其他属性（CSS3）：content/cursor/border-radius/box-shadow/text-shadow

    定位（定位模式+边偏移）

    • 定位模式：static、relative、absolute、fixed
    • 边偏移：top、bottom、left、right
    • 静态定位：默认定位方式，无定位的意思 position: static;
    • 相对定位：根据它原来的位置移动，但还保持原来的位置 position: relative;
    • 绝对定位：如果没有祖先元素或者祖先元素没有定位，则以浏览器为准
      如果祖先元素有定位（相对、绝对、固定），则以最近一级有定位的元素为参考，绝对定位不占有原来的位置 position: absolute;
    • 子绝父相的由来：父级需要占有位置，因此是相对定位，子盒子不需要占有位置，则是绝对定位
    • 固定定位：以浏览器的可视窗口为参照点移动，不占有原先的位置 position: fixed;
    • 固定定位小技巧：固定在版心右侧位置 1. 让固定定位的盒子 left: 50%;，走到浏览器可视区的一半位置
    1. 让固定定位的盒子 margin-left: 版心宽度的一半;
    • 粘性定位：以浏览器的可视窗口为参照点移动，占有原先的位置，必须添加 top、left、right、bottom其中一个才有效 position: sticky; top: 10px;
    • 定位叠放次序 z-index: auto(默认)/正整数/负整数/0;
      定位叠放次序注意事项：1. 数值越大，盒子越靠上 2. 如果属性值相同，按照书写顺序，后来居上 3. 只有定位的盒子才有 z-index
    • 加了绝对定位的盒子不能通过 margin: 0 auto; 水平居中，可以通过 1. left: 50%; 2. margin-left: -自身宽度的一半;
    • 定位特殊特性（与浮动类似）：1. 行内元素添加绝对定位或者固定定位（因为绝对定位和固定定位都不占有原来的位置），可以直接设置高度和宽度 2. 块级元素添加绝对或者固定定位，如果不给高度或者宽度，默认大小是内容的大小（正常情况块级元素默认宽度是父元素的百分之百，高度是内容的高度） 3. 浮动元素、绝对或固定定位元素都不会触发外边距合并的问题 4. 与浮动元素不同，只会压住它下面的标准流盒子，但是不会压住下面标准流盒子里面的文字（图片），但是绝对定位（固定定位）会压住下面标准流所有的内容

    显示与隐藏

    • display 隐藏元素后，不再占有原来的位置
    • display: none; 隐藏对象
    • display: block; 除了转换为块级元素外，同时还有显示元素的意思
    • visibility 用于指定一个元素可见还是隐藏
    • visibility: visible; 元素可视
    • visibility: hidden; 元素隐藏，继续占有原来的位置
      overflow 溢出
    • overflow: visible; 默认
    • overflow: hidden; 溢出隐藏，继续占有位置
    • overflow: scroll; 溢出不溢出都显示滚动条
    • overflow: auto; 溢出的时候才显示滚动条，不溢出不显示滚动条

    扩展

    • 鼠标样式 cursor: default(默认)/pointer(小手)/move(移动)/text(文本)/not-allowed(禁止);
    • 取消表单轮廓线 outline: 0/none; 可以去掉默认的边框
    • 防止拖拽文本域 resize: none;
    • vertical-align: baseline(默认)/top/middle/bottom; 用于设置一个元素的垂直对齐方式，它只针对于行内元素或行内块元素
    • 解决图片底部默认空白缝隙问题（因为默认行内块元素和文字基线对齐）：● 给图片添加样式 vertical-align: middle/top/bottom; ● 把图片转换为块级元素 display: block;
    • 溢出的文字省略号显示：
      ● 单行文本溢出显示省略号：1. 先强制一行内显示文本 white-space: nowrap; 2. 超出的部分隐藏 overflow: hidden; 3. 文字用省略号替代超出的部分 text-overflow: ellipsis;
      ● 多行文本溢出显示省略号：1. overflow: hidden; 2. text-overflow: ellipsis; 3. display: -webkit-box; 弹性伸缩盒模型显示 4. -webkit-line-clamp: 2; 限制在一个块元素显示的文本的行数 5. -webkit-box-orient: vertical; 设置或检索伸缩盒对象的元素的排列方式
