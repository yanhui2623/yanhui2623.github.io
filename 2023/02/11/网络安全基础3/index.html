    用户管理

    用户概述
    每一个用户登录系统后，拥有不同的操作权限
    每个账号有自己唯一的 SID（安全标识符 = 系统 ID + 用户 ID）
    用户 SID：S-1-5-21-426206812-2579496042-14852678-500（查询命令 whoami /user）
    系统 SID：S-1-5-21-426206812-2579496042-14852678
    Windows 系统管理员 administrator 的 UID 是 500
    普通用户的 UID 是 1000 开始
    不同的账户拥有不同的权限，为不同的账号赋予权限，也就是为不同账户的 SID 赋予权限
    账号密码存储位置：c:\windows\system32\config\SAM
    windows 系统上，默认密码最长有效期 42 天

    内置账户
    给人使用的账户：administrator 管理员账户、guest 来宾账户
    计算机服务组件相关的系统账号：、system 系统账户（权限至高无上）、local services 本地服务账户（权限等于普通用户）、network services 网络服务账户（权限等于普通用户）
    权限大小：system > administrator > local services = network services > guest

    配置文件
    每个用户都有自己的配置文件（Linux 的 home 目录），在用户第一次登录时自动产生，路径是：
    win 7/win 2008 c:\用户\
    xp/win 2003 c:\Documents and Settings\
    实操：点击我的电脑并右键点击管理，找到本地用户和组并点击，可以看到三个账户，分别是 administrator、guest、SUPPORT_388945a0（此文件可以删除没有影响），在空白处右键可以创建新用户

    常用命令
    net user 列出当前账户列表
    net user 用户名 当前用户的详细信息
    net user 用户名 新密码 给用户更改密码（只有管理员可以更改），如果没有这个用户名则显示找不到用户名
    net user 用户名 密码 /add 新建用户（每个新建的用户都属于 Users 组，只有读取和运行、列出文件夹目录、读取权限、特别的权限）
    net user 用户名 /del 删除用户
    net user 用户名 /active:yes 激活或禁用账户（yes 或 no）

    组管理

    组的作用：简化赋予的权限
    赋权限的方式：用户——组——赋权限、用户——赋权限
    内置组：内置组的权限默认已经被系统赋予 administrators管理员组、guests来宾组、users普通用户组、network网络配置组、print打印机组、Remote Desktop远程桌面组
    net localgroup 列出当前组列表
    net localgroup administrators 列出当前组的成员
    net localgroup CEO /add 添加新的组 CEO
    net localgroup administrators shimisi /add 添加 shimisi 到管理员组
    net localgroup administrators shimisi /del 删除管理员的 shimisi 账号
    实操：点击我的电脑并右键点击管理，找到本地用户和组并点击，在点击组即可查看

    服务器远程登录

    远程管理类型
    Windows 远程管理有两种类型：远程桌面（图形）、telnet（命令行）

    远程桌面（图形）步骤

    1. 首先将配置网络，并实现客户机与服务器可以互通（连接在一个局域网下即可）
    2. 服务器开启允许被远程控制：点击计算机（xp 或 2003 是点击我的电脑）并右键点击属性——远程设置——选择允许——确定
    3. 客户机上：win + R——输入 mstsc 打开远程连接工具
    4. 在 mstsc 工具上输入服务器的 IP 并点击确定
    5. 输入服务器的账号及密码
      注意：如果使用非管理员账户登录远程，需要在服务器上将用户加入到远程桌面内置组Remote Desktop Users 中 net localgroup "Remote Desktop Users" 用户名 /add 如果是第一次登录会进行初始化

    telnet（命令行）步骤

    1. 首先将配置网络，并实现客户机与服务器可以互通（连接在一个局域网下即可）
    2. win + R 输入 services.msc 打开服务，找到 Telnet 服务并双击，启动类型改为自动，再点击应用并确定即可
    3. win + R 输入 cmd 打开终端 telnet 目标IP地址，输入账户密码，如果不是管理员账户会显示失去了跟主机的连接，说明该账户权限不够 net localgroup "TelnetClients" 用户名 /add 重新登录即可成功

    netstat -an 可以查看本机开放的所有端口（23 telnet协议 3389 远程桌面协议RDP）

    破解 win 7 系统密码

    注意：win 7 和 win 10 系统已修补漏洞，则无法利用，cmd 工具路径：c:\windows\system32\cmd，用户账号密码存储位置：c:\windows\system32\config\SAM，修改账户密码命令：net user 用户名 新密码

    1. 在账户登录页面连续按五次 shift 键，会弹出一个粘滞键窗口（这个程序路径是 c:\windows\system32\sethc.exe）此操作验证系统是否有这个漏洞，部分 win 7 及 win 10 系统在未进入系统时，可以通过系统修复漏洞篡改系统文件名
    2. 右下角关机再次开机在正在启动 windows 页面看见有windows 图标出来 点击帮助右边的按钮下拉菜单选择重新启动客户机，如果出现 “启动修复（推荐）” 及选项，如果没出现多尝试几次
    3. 出现启动修复的对话框，有还原和取消两个选项，选择取消系统会自动修复等待几分钟后会弹出新的窗口，点击隐藏问题详细信息，将滚动条翻到最下面点击最后一个记事本的连接打开时一个 txt 文件
    4. 点击文件再点击打开会显示所有文件（这时的 D 盘是 C 盘，因为 100M 的系统文件被当作了 C 盘，但是在 system32 目录下找不到 sethc.exe，因为当前显示的都是文件夹当然找不到，只要将下面的文件类型改为所有文件即可找到）找到 sethc（蓝白色图标） 并改名 sethc-bak（任意英文名），再找到 cmd，复制一份 cmd 并粘贴再改名为 sethc，关闭三个窗口剩余一个窗口点击完成
    5. 系统启动完毕后，连续按 5 次 shift 键，将弹出 cmd 工具，使用 net user 用户名 "" 命令可以设置空密码或者将以前的密码改掉，或者另外建立一个用户，net user 新用户 密码 /add，并提升为管理员，net localgroup administrators 新用户 /add，重启一下新用户就生效了，注销后再删除新建的用户，net user 新用户 /del这样的好处是不修改当前用户的密码即可登录系统
